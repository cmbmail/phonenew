银行内部电话费用分摊系统

需求文档 & 设计方案

基于电信账单的多级组织费用分摊与确认系统

V1.1  |  2026-06-22  |  密级：内部  |  状态：评审中
PRD技术架构DDLAPI

一、项目概述

1.1 项目背景

银行内部使用中国电信提供的电话服务，每月产生大量通讯费用（基础通话费、录音费、彩铃费、闪信费）。这些费用需要按号码归属逐级分摊到集团→一级分行→二级分行→部门，供各级财务入账。

当前依赖人工Excel操作，效率低、易出错、难以追溯。本系统旨在实现导入电信账单→自动匹配号码归属→按组织层级分摊→逐级确认→导出分行账单的全流程自动化。

1.2 5W1H分析

维度内容
Why人工分摊效率低、错误率高、追溯困难
What基于电信账单的多级组织费用分摊与确认系统
Who4类角色：集团管理员、分行管理员、部门管理员、财务人员
When每月账单周期T+5日前完成导入分摊，T+15日前完成确认
Where银行内部Web系统，独立部署
How导入账单→匹配归属→自动分摊→逐级确认→导出Excel

1.3 核心业务流

号码归属表导入→ 通讯录导入→ 归属校验→ 借调+例外处理→ 账单导入→ 自动分摊→ 逐级确认→ 导出Excel

1.4 系统边界

  • 运营商：仅中国电信（单一运营商）
  • 费用类型：基础通话费、录音费、彩铃费、闪信费（4类）
  • 无公共费用分摊：所有费用均可归属到具体号码
  • 无审批流程：逐级确认即可，不需要审批
  • 无财务系统API对接：通过Excel导出实现

三、数据源与导入

3.1 三份源数据文件

文件列数关键字段用途
号码归属.xlsx2列外线号码+描述分行级归属+例外标记+通讯录缺失时的兜底
通讯录.xlsx4列部门全路径+用户名+分机+外线主归属源：号码→员工→部门（最细粒度）
电信账单.xlsx4 Sheets按号码费用/录音/彩铃/闪信费用数据源，按Sheet分别解析

3.2 号码归属表

格式：2列（外线号码 + 描述）

描述字段用 / 分隔组织层级，层数不固定。描述以 [例外] 前缀开头的号码，优先级最高（P0）。

外线号码描述解析结果
01056089999北京分行/营业部归属北京分行营业部
01088881234[例外]总行专用P0 例外

3.3 通讯录

格式：4列（部门全路径 + 用户名称 + 分机号码 + 外线号码）

部门全路径用 - 分隔，层数不固定。用户名称为纯数字（=员工ID）。

部门全路径包含借调/挂职/交流/代管/外派/轮岗/驻点等关键词时，自动标记为借调。

部门全路径用户分机外线检测
总行-信息科技部-软件开发中心张三36770201056089999
北京分行-运营管理部(借调)李四36770301056080001借调

3.4 电信账单

4个Sheet结构，Sheet名含月前缀（如"2026年3月按号码费用"），通过sheetNamePattern正则匹配。
Sheet列数号码列位置费用字段
按号码费用9列A列平台使用费+码号月租费+国内外呼时长+转接外呼时长+国内费用+国际时长+国际费用+费用小计+备注
录音4列B列分机号+号码+录音目录+费用小计
彩铃3列B列分机号+号码+费用
闪信4列A列号码+月份+下发量+金额（季度结算）

闪信季度结算：所有月份的闪信数据累计到当前月份总额，按号码归属分摊。

3.5 导入顺序（关键依赖）

号码归属表→ 通讯录→ 归属校验→ 借调+例外→ 账单导入→ 分摊→ 确认→ 导出

3.6 账单模板配置（bill_template）

采用JSON配置驱动解析，支持不同月份账单格式微调：

  • sheetNamePattern：正则匹配Sheet名（如 /按号码费用$/）
  • phoneColumn：号码所在列（A或B，不同Sheet位置不同）
  • feeMappings：费用列到字段的映射（如 {F:'call_fee', H:'international_fee'}）
  • isQuarterly：是否季度结算（闪信=true）
  • skipRows：跳过行数（表头行）
{ "sheetNamePattern": "按号码费用$", "phoneColumn": "A", "feeMappings": { "E":"monthly_rent_platform","F":"monthly_rent_code","J":"call_fee_domestic","K":"call_fee_international","L":"total_fee" }, "isQuarterly": false, "skipRows": 1 }

四、号码归属优先级与匹配

4.1 三源归属模型

优先级来源标记含义
P0号码归属表is_exception=1例外号码，最高优先级，不可被任何规则覆盖
P1通讯录部门全路径主归属源，最细粒度（到员工/部门）
P2号码归属表描述字段兜底归属，通讯录缺失时使用（仅到分行级）
P3无UNASSIGNED两个源均无该号码，待人工分配

4.2 匹配算法

function resolveOwnership(phoneNumber): if 号码归属表.is_exception(phoneNumber): return { source: 'P0', org: 归属表.org, is_exception: true } if 通讯录.has(phoneNumber): return { source: 'P1', org: 通讯录.department, is_exception: false } if 号码归属表.has(phoneNumber): return { source: 'P2', org: 归属表.branch, is_exception: false } return { source: 'P3', org: NULL, is_exception: false }

4.3 借调处理

默认规则：费用归属到编制部门（从通讯录部门全路径解析）

自动检测：部门全路径包含借调/挂职/交流/代管/外派/轮岗/驻点关键词 → 标记 is_seconded=true

人工调整：管理员可将借调人员费用转移到实际工作部门，调整记录存入 allocation_adjustment 表

五、分摊与确认机制

5.1 自动分摊规则

分摊逻辑：按号码归属到组织。每个号码的费用归入其所属组织，并沿组织树向上汇总。

  • 每个号码 → 归属到末级组织（部门）
  • 部门费用 = 该部门所有号码费用之和
  • 分行费用 = 该分行下属所有部门费用之和
  • 集团费用 = 所有分行费用之和 = 总账单金额（校验条件）

5.2 费用字段（6项）

字段来源说明
monthly_rent平台使用费+码号月租费月租类固定费用
call_fee国内费用+国际费用通话类变动费用
recording_fee录音Sheet费用小计录音增值业务费
crbt_fee彩铃Sheet费用彩铃增值业务费
flash_msg_fee闪信Sheet金额闪信增值业务费（季度结算）
total_fee以上5项之和费用合计

5.3 批次状态机

状态名称说明
0DRAFT草稿：账单已导入，分摊数据待校验
1ALLOCATED已分摊：分摊完成，待逐级确认
2CONFIRMED已确认：所有层级确认完成
3LOCKED已锁定：已进入财务入账流程，不可撤回

5.4 逐级确认与撤回

集团确认→ 一级分行确认→ 二级分行确认→ 部门确认→ LOCKED
  • 撤回需填写原因，级联撤回所有下级
  • 已锁定（LOCKED）状态不可撤回
  • 所有变更操作记录审计日志

六、分行账单导出

6.1 导出格式（基于银行财务系统模板）

关键变更：导出格式已对齐银行财务系统实际导入模板（V2），不再是V1设计稿中的5 Sheet复杂格式。

文件1：分行费用分摊汇总.xlsx

按成本中心汇总费用，供财务系统批量导入入账。

#列名示例说明
A成本中心121006末级部门成本中心代码（数值型）
B费用小计(单位：元)11.615该成本中心全部费用合计

文件2：分行费用分摊明细.xlsx

按号码列示明细，供分行核对和追溯。

#列名示例说明
A号码085199911027电信外线号码（字符串型）
B成本中心121006号码归属的成本中心代码
C费用小计(单位：元)11.62该号码全部费用合计

核心规则

  • 成本中心 = sys_organization.code（末级部门代码，如121006）
  • 费用小计 = total_fee（6项费用之和）
  • 汇总：按成本中心聚合，金额保留3位小数
  • 明细：按号码列示，金额保留2位小数
  • 不含P3未归属号码
  • 完整性校验：汇总SUM = 明细SUM = 总账单金额

6.2 V1→V2 变更对比

V1 设计稿（已废弃）

  • 1个Excel（5个Sheet）
  • 汇总表13列（分行+7项费用+合计）
  • 4个明细Sheet按费用类型拆
  • 组织标识：一级分行+部门代码+部门名称
  • 费用按月租/通话/录音/彩铃/闪信拆分

V2 实际模板（当前）

  • 2个Excel（各1个Sheet）
  • 汇总2列（成本中心+费用小计）
  • 1个明细Sheet（3列，不拆费用类型）
  • 组织标识：成本中心（=部门代码）
  • 费用仅合计（total_fee）
设计取舍：导出文件面向财务系统（极简），费用构成拆分通过系统页面交互式满足（可钻取、可筛选、可对比）。

US-001 导入号码归属表

作为集团管理员，我想要导入号码归属表，以便建立号码与组织的归属关系。

  • 自动识别[例外]前缀并标记is_exception=1
  • 描述字段用"/"分隔解析组织层级

US-002 导入通讯录

作为集团管理员，我想要导入通讯录，以便建立号码→员工→部门的归属链。

  • 解析4列数据，部门全路径用"-"分隔
  • 自动检测借调关键词并标记is_seconded=true
  • 与号码归属表交叉校验，输出P0-P3分布报告

US-003 导入电信账单

作为集团管理员，我想要导入电信账单，以便开始费用分摊流程。

  • 支持4个Sheet分别解析，Sheet名通过模板正则匹配
  • 不同Sheet号码列位置不同（A/B）自动识别
  • 闪信Sheet标记isQuarterly=true，季度累计

US-004 号码归属校验

作为集团管理员，我想要校验号码归属，以便确认P0-P3分布并处理未归属号码。

  • 对所有号码执行P0→P1→P2→P3匹配
  • 输出校验报告：各优先级数量和占比
  • P3未归属号码可人工分配组织

US-005 自动分摊

作为集团管理员，我想要系统自动按号码归属分摊费用到组织。

  • 沿组织树向上汇总到分行和集团
  • 集团合计=总账单金额（数据完整性校验）

US-006 借调调整

作为分行管理员，我想要调整借调人员的费用归属，使费用归入实际工作部门。

  • 选择借调号码，指定目标部门
  • 调整后自动重新计算分摊结果
  • 调整记录写入审计表

US-007 逐级确认

作为各级管理员，我想要确认本级的分摊结果，以便完成费用入账。

  • 集团确认→自动生成分行分摊数据
  • 撤回需填写原因，级联撤回下级
  • LOCKED状态不可撤回

US-008 导出分行分摊Excel

  • 格式严格匹配银行财务系统模板（成本中心+费用小计）
  • 导出前执行完整性校验

11.4 分摊引擎算法

输入: bill_batch_id + 全部 bill_detail 记录 ─────────────────────────────────────── 1. 批量加载 bill_detail (按 batch_id) 2. 遍历每条 bill_detail: → 按 org_id 汇总到 allocation_result 3. 沿组织树向上汇总: 部门 → 二级分行 → 一级分行 → 集团 4. 校验: 集团 total_fee = SUM(bill_detail.total_fee) 5. 更新 bill_batch.status = ALLOCATED 6. 批量写库: saveAll(batch_size=100) ─────────────────────────────────────── 优化: 组织树一次性加载并缓存, 避免N+1

11.5 关键技术决策

决策点选择理由

分摊算法纯内存+批量写1万条明细内存聚合极快，避免逐条写库
Excel导出后端POI生成极简2+3列格式，POI轻松胜任

十三、非功能需求

性能

  • 账单导入（含4 Sheet、1万行明细）< 30秒
  • 自动分摊计算 < 10秒
  • 分行分摊导出（汇总+明细2个文件）< 10秒
  • 页面首次加载 < 3秒，路由切换 < 500ms

可靠性

  • 导入操作幂等：重复导入同一批次不产生重复记录
  • 分摊结果与总账单金额强一致：校验失败不允许确认
  • 数据库事务保证：导入、分摊、确认操作原子性

安全性

  • JWT认证 + RBAC权限控制
  • 所有API接口参数校验，防SQL注入/XSS
  • 敏感信息环境变量化，日志脱敏
  • 操作审计：所有变更操作记录审计日志

可扩展性

  • 账单模板JSON配置：新增Sheet/字段无需改代码
  • 借调关键词可配置：无需改代码
  • 组织层级支持扩展：当前4级，可扩展到N级

兼容性

  • 浏览器：Chrome 90+、Edge 90+、Firefox 88+
  • Excel：导出文件兼容Microsoft Excel 2016+、WPS 2019+
